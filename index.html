CLAUDE CODE

Custom Agent System

本構成は、単なる設定ファイルの集合ではありません。記憶を保持し、文脈を自動で取捨選択し、専門エージェントを呼び出し、自らを点検する「6層のAIエージェント組織」として動作します。各タブから、その全体像をご覧いただけます。

// Overview

What Is This?これは何か — ひとことで

会話のたびに、この6つの層が連携して動作します。上の層ほど土台・記憶を、下の層ほど実行・判断を担います。中でも3番目の層が中核で、一般的な設定と最も異なる部分です。

覚えている

過去の会話の事実や決定事項をファイルに残し、次回も参照します。

文脈を取捨選択する

現在の話題に関連する情報のみを手前に展開し、不要なものは退避します。

専門家を呼び出す

必要なときだけ、得意分野ごとの専門AIに作業を委任します。

自らを点検する

設定の不整合を定期的に検査し、人間の確認が必要な操作では停止して問い合わせます。

// The Six Layers

Six Layers6つの層 — それぞれの役割

上から下へ、土台 → 記憶 → 自動制御 → 実行 → 運用 → 判断の順に積み上がっています。

LAYER 01

静的設定

毎回読む土台・ルール

会話のたびに必ず読み込む土台です。実行可能な操作・応答方針・進め方のルールを定義します。大きな一覧は「必要なときだけ読む」方式に切り替え、常時の読み込み負荷を42%削減しました。

CLAUDE.md（目次）AUTHORITY（判断基準）委任ルール応答方針
LAYER 02

記憶

会話をまたいで覚える

会話が終了しても保持される記憶です。「利用者の好み」「作業方針」「進行中の案件」「外部リソース」の4種類で、コードからは読み取れない事実を蓄積します。

索引（MEMORY.md）利用者情報作業方針案件メモ
中核
LAYER 03

自動で動くしくみ

関連情報の出し入れ・協調

本システムが一般的な設定と最も異なる中核部です。会話のたびに自動実行される小さなプログラム（フック）が、話題に関連する情報のみを記憶の手前へ展開します。複数のインスタンスが同時に動作する際は、互いの状況を共有します。

関連情報の自動展開インスタンス協調学習ログ
LAYER 04

実行

作業を担う

実際の作業を担う層です。定型手順をこなす「スキル」が17個、得意分野ごとの「助っ人AI」が144体。現在主に使う技術（TypeScript・React 等）に一致する助っ人は、確認なしで起動できます。

スキル 17助っ人AI 144役割の4階層
LAYER 05

運用・自己点検

健全性を保つ

「AIの仕組みは警告なく静かに壊れる」という課題への対策層です。作業を看板で可視化し、成果物は2名体制でレビュー、月次で不要物を整理し、設定の健康診断を定期実施します。

看板ボード2名レビュー健康診断月次整理
LAYER 06

判断のルール

自律の範囲を定める

全層を貫く責任の線引きです。AIが自ら決定してよい操作・人間に確認すべき操作・絶対に行わない操作を、信号機の色（緑／黄／赤）で区分します。詳細は「判断のルール」タブをご覧ください。

緑 実行可黄 要確認赤 禁止

// How It Works

How a Chat Flows1回の会話で起きること

メッセージが送信されるたびに、裏側で4つの自動スイッチ（フック）が順に働きます。

会話の流れ

左から右へ。各ステップで自動スイッチが1つずつ発火します。

関連情報の出し入れ（注意のしくみ）

記憶を常に全展開すると処理の妨げになります。そこで「現在どれだけ関連するか」で3段階に分け、展開量を調整します。よく使う話題は減衰せず、関連ファイルは互いの優先度を引き上げます。

よく使う作業中の話題 — 全文を展開
たまに使う見出しのみ展開
使っていない退避（しまう）

複数で協調する（共有メモ）

複数のインスタンスが同時に動作する際、互いの状況を1つの共有メモに書き込み、競合を防ぎます。次の3つのタイミングで自動更新されます。

はじまり：直近の共有メモを読み込む 入力ごと：状況を追記する 終わり：要点を残す

// External Connections (MCP)

7 Integrations外部連携（MCP）— 外部サービスへの接続口

MCP（Model Context Protocol）は、AIを外部のツールやデータに接続する仕組みです。以下のサービスを claude.ai 経由で接続しています。

接続中のMCPサーバー7

context7
ライブラリやフレームワークの最新公式ドキュメントを取得する

supabase
Supabase（データベース・バックエンド）を参照・操作する
Canva
デザインの生成・編集・書き出しを行う
Gmail
メールの検索・スレッド取得・下書き作成を行う
Google Calendar
カレンダーと連携する
Google Drive
ドライブ上のファイルを検索・読み込みする

※ いずれも claude.ai 経由のクラウド連携。読み取りは確認不要、書き込み・更新系は「判断のルール」に従い確認のうえ実行します。

// Authority

Traffic Lights判断のルール — 自律の範囲

AIの暴走を防ぐための線引きです。信号機の色で「自ら決定してよい／確認のうえ実行／絶対に行わない」を区分します。判断に迷う場合は、必ず黄（確認）に倒します。

緑

GREEN · 無確認で実行可

自らの判断で実行し、完了後に1行で報告します。

  • ファイルの読み取り・検索
  • 看板の更新
  • 記憶の追加

黄

YELLOW · 要確認

選択肢を2〜4個提示し、いったん停止して確認します。

  • 設定ファイルの編集
  • プロジェクト本体の変更
  • ファイルの削除

赤

RED · 絶対禁止

いかなる状況でも実行せず、依頼があっても理由を返して停止します。

  • 認証情報の送信
  • 課金・支払い
  • 本番データの破壊
自身の作業範囲のみ
共有・設定・プロジェクト
やり直せる可逆
緑そのまま実行し報告
黄選択肢を提示し確認
やり直せない不可逆
黄選択肢を提示し確認
赤実行しない

※「常に赤」の特例（認証情報の送信・課金・本番破壊・外部への意図しない書き込み）は、この表より先に判定されます。
